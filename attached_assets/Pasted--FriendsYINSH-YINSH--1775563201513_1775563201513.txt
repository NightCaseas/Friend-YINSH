# Вижен продукта: FriendsYINSH

## Проблема
Фанат абстрактной стратегии YINSH не может сыграть партию, потому что в его ближнем круге нет соперника, а физическая встреча с друзьями требует синхронизации времени и места. Существующие онлайн-решения либо отсутствуют, либо перегружены регистрацией и лишними функциями.

## Целевой пользователь
Мужчина или женщина 30–40 лет, любитель абстрактных настольных стратегий. Уверенно пользуется браузером, умеет открывать ссылки из мессенджеров. **Не хочет регистрироваться**, запоминать пароли или разбираться в интерфейсе. Ценит простоту: «перешёл по ссылке — сразу играю».

## Job-to-be-done
Когда у пользователя появляется желание сыграть в YINSH и есть конкретный друг, готовый составить партию, он «нанимает» продукт, чтобы за 10 секунд запустить игру. Контекст: переписка в Telegram или WhatsApp → кинул ссылку → друг перешёл → начали ходить. Без ожидания очереди, без рейтинга, без ботов.

## Решение в одном предложении
**FriendsYINSH** — это веб-приложение для двух игроков, которое позволяет сыграть партию в YINSH по прямой ссылке без регистрации.

## Ключевые сценарии

1. **Создание игры**  
   Пользователь заходит на сайт → нажимает «Создать игру» → получает уникальную ссылку (например, `site.com/game/abc123`) → копирует и отправляет другу.

2. **Подключение к игре**  
   Друг открывает ссылку → система автоматически определяет роли: создатель играет белыми, подключившийся — чёрными → игра начинается сразу.

3. **Ход игры и завершение**  
   Игроки по очереди делают ходы (клик по своему кольцу → клик по доступной клетке). При захвате 3 колец игра заканчивается, показывается экран победителя. В любой момент доступна кнопка «Сдаться», которая немедленно завершает партию победой соперника.

## MVP-скоуп

**Что входит:**
- Игровое поле 8×8 с расстановкой колец по правилам YINSH
- Подсветка доступных ходов при клике на своё кольцо
- Переключение ходов между игроками
- Проверка победы (3 захваченных кольца)
- Кнопка «Сдаться»
- Создание комнаты с уникальной ссылкой
- Подключение по ссылке
- Автоматическое назначение сторон (создатель — белый, подключившийся — чёрный)

**Что НЕ входит:**
- Регистрация, логины, почта
- Бот или ИИ
- Очередь и матчмейкинг
- Таймер на ход
- Сохранение партий и статистика
- Чат, анимации, звуки
- Игра с произвольным соперником (только по ссылке)

## Критерий успеха демо (через 2 дня)

Тест, который подтвердит, что продукт «получился»:
> Открыть две вкладки браузера. В первой создать комнату, во второй перейти по ссылке из первой. Сделать ход белыми в первой вкладке — ход мгновенно отображается во второй вкладке. Продолжить игру до победы одного из игроков (3 захваченных кольца). Экран победителя появляется корректно.

Если это работает — продукт успешен. Всё остальное — улучшения после тренинга.

## Риски и допущения

1. **Риск: синхронизация состояния игры.** Без сервера с WebSocket или Firestore два клиента могут рассинхронизироваться (один сделал ход, второй не увидел). Допущение: используем Firebase Realtime Database или простой WebSocket-сервер (например, Colyseus или прямой `onSnapshot` в Firestore) — это реализуемо за 2 дня.

2. **Риск: правила YINSH сложны в отладке.** Проверка захвата колец и построения линий — нетривиальная логика. Допущение: мы закодируем только базовые правила без учёта всех крайних случаев (например, двойной захват), и на демо это не взорвётся.

3. **Риск: два игрока могут попытаться ходить одновременно.** Без блокировки хода возможны конфликты. Допущение: на первых порах делаем простой флаг «чей ход» на сервере, и клиент с не своим ходом просто не отправляет ходы.
